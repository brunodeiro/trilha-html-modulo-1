Clients x Servers

(Voltar)

Clients: São os responsáveis por fazer as solicitações e utilizações de serviços ou recursos. Exemplos de clients são todos os dispositivos de computação, como desktops, notebooks, smartphones, tablets..

Servers: É a parte responsável em armazenar e fornecer os serviços, recursos ou dados solicitados pelos clients.

A comunicação client-server é baseada em troca de mensagens, onde o client envia requisições e o server responde com as informações requisitadas. Desta forma podendo ser implementada nas mais diversas formas, como por exemplo em aplicações web, sistemas de gerenciamento de bancos de dados (sgbd), jogos, dentre outros.

O que é HTML?

(Voltar)

Diferente do que muitos podem pensar HTML não é uma linguagem de programação. Seu significado vem de Linguagem de Marcação de HiperTexto (Hypertext Markup Language) na qual sua utilização é voltada na construção de paginas na web. Foi desenvolvida pelo professor e fisico Tim Berners-Lee com o intuito de estabelecer uma estrutura basica para website.

Entendendo a estrutura básica de um HTML

(Voltar)

Podemos dividir a estrutura de um html em 03 partes, sendo elas:

  • HTML: Define o documento HTML
  • HEAD: Define o cabeçalho da página (Não aparece para o usuário)
  • BODY: Define o corpo do documento HTML (será mostrado ao usuário)

Listas: Ordenadas x Não Ordenadas

(Voltar)

ORDENADAS: São listas numeradas criadas para ordenação.
Exemplo:

  1. Posição A
  2. Posição B
  3. Posição C

NÃO ORDENADAS: São listas que não possuem numeração alguma.
Exemplo:

  • Posição A
  • Posição B
  • Posição C
